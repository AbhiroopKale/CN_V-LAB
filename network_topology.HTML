Theory

1. What is Network Topology?

Network topology refers to the arrangement of nodes (computers, routers, switches) and links (cables, wireless connections) in a network. It defines how devices are interconnected and how data flows between them.

Topology is classified into two types:

  • Physical Topology — the actual physical layout of cables and devices in the network.
  • Logical Topology — the way data actually travels through the network, regardless of physical arrangement.

Choosing the right topology affects network performance, cost, reliability, and scalability. There is no single "best" topology — the optimal choice depends on the application, budget, and scale.

2. Bus Topology

In a Bus topology, all nodes are connected to a single shared communication line called the backbone or bus. Data transmitted by any node travels in both directions along the bus and is received by all nodes — only the intended recipient processes it.

  • Terminator resistors are placed at both ends to prevent signal reflection.
  • Uses CSMA/CD (Carrier Sense Multiple Access with Collision Detection) to manage access.
  • A break anywhere in the backbone cable brings down the entire network.
  • Historically used in 10BASE2 (Thinnet) Ethernet networks.

Advantages: Inexpensive, easy to install, works well for small networks.
Disadvantages: Single point of failure on the backbone, performance degrades with more nodes, difficult to troubleshoot.

3. Star Topology

In a Star topology, all nodes are connected to a central device — typically a switch or hub. All communication between nodes passes through this central point.

  • If a node fails, only that node is affected; the rest of the network continues normally.
  • If the central switch fails, the entire network goes down.
  • Easy to add/remove nodes without disrupting the network.
  • Most widely used topology today — found in Ethernet LANs (100BASE-TX, Gigabit Ethernet).

Advantages: Easy troubleshooting, fault isolation, scalable.
Disadvantages: Higher cable cost, central device is a single point of failure.

4. Ring Topology

In a Ring topology, each node is connected to exactly two other nodes, forming a closed circular loop. Data travels in one direction (unidirectional) or both directions (bidirectional / Dual Ring) around the ring.

  • Uses a token-passing protocol — only the node holding the token may transmit data.
  • A single break in a unidirectional ring can disable the entire network.
  • Dual Ring (e.g., FDDI) provides redundancy — if one ring fails, the other takes over.
  • Used in Token Ring (IEEE 802.5) and FDDI (Fiber Distributed Data Interface).

Advantages: Equal access, predictable performance, no collisions with token passing.
Disadvantages: A node failure can break the ring, slower than star for modern networks.

5. Mesh Topology

In a Mesh topology, every node is connected to every other node, either fully (full mesh) or partially (partial mesh). This provides maximum redundancy and fault tolerance.

  • Full Mesh: n nodes require n(n–1)/2 links — very expensive for large networks.
  • Partial Mesh: Only critical nodes have multiple connections, reducing cost.
  • Data can take multiple paths — if one link fails, traffic is rerouted automatically.
  • Used in WAN backbone networks, the Internet core, and military networks.

Advantages: Highest fault tolerance, multiple data paths, no single point of failure.
Disadvantages: Very expensive, complex to install and manage.

6. Tree (Hierarchical) Topology

A Tree topology is a hierarchical arrangement that looks like a tree with branches. It extends the Star topology by connecting multiple star networks to a common root bus or backbone.

  • Has a root node, intermediate nodes (switches), and leaf nodes (end devices).
  • Organised in levels — data flows up and down the hierarchy.
  • Used in campus LANs, corporate networks, and cable TV (CATV) systems.
  • If a branch switch fails, all nodes below it lose connectivity.

Advantages: Scalable, organised, easy to manage sections independently.
Disadvantages: Backbone failure is catastrophic; complex wiring.

7. Hybrid Topology

A Hybrid topology combines two or more different topologies into a single network. Real-world enterprise networks almost always use a hybrid design to balance cost, performance, and redundancy.

  • Example: A Star-Bus hybrid connects multiple star topologies along a common backbone bus.
  • Another example: Star-Ring connects star-configured workgroups in a ring at the core.
  • Provides flexibility — different departments can use the topology best suited to their needs.

Advantages: Flexible, scalable, fault tolerant when designed well.
Disadvantages: Complex to design, configure, and maintain.

Procedure

Follow these steps to perform the network topology experiment using the Simulation tab.

Simulation Overview

The Simulation tab lets you select any of 6 topologies, observe their structure, send a packet between two nodes, trigger node or link faults, and watch how each topology responds. Each topology is drawn live on the canvas.

1

Select a Topology

Open the Simulation tab. Use the topology selector buttons (Bus / Star / Ring / Mesh / Tree / Hybrid) to choose a topology. The canvas will render the selected topology immediately.

Observe: Note the number of nodes, links, and layout pattern. Record the structure in your observation sheet.
2

Inspect Topology Statistics

Check the Info Panel on the right side. It shows the number of nodes, total links, links per node, and fault tolerance rating for the selected topology.

Record: For each topology, note the values — compare how Mesh has far more links than Bus for the same number of nodes.
3

Send a Packet

Click "▶ Send Packet". An animated packet will travel from a source node to a destination node, following the path dictated by the topology. Watch the path highlight on the canvas.

Observe: In Bus topology, the packet travels the entire bus. In Star, it goes through the central hub. In Ring, it hops node-by-node. Log panel shows each hop.
4

Trigger a Fault

Click "⚡ Simulate Fault" to randomly disable a node or link. The canvas will highlight the failed component in red. Then click "Send Packet" again to see how the network responds.

Record: Which topologies allow delivery even after a fault? (Mesh, Hybrid — yes. Bus, Ring — often no.) Document the result for each topology.
5

Repeat for All Topologies

Repeat steps 1–4 for all 6 topologies. Use "↺ Reset" before switching if a fault was introduced. Fill in your observation table with the recorded data.

6

Compare and Conclude

Using your observations, answer: Which topology is best for a small office? A hospital (needing high redundancy)? A school campus? Relate your answers to the Theory section.

Expected Conclusion: Star for ease of management, Mesh for critical systems, Tree for large campuses, Hybrid for enterprise environments.

Result

Six fundamental network topologies were studied and simulated. Each topology was analysed for its structure, packet routing behaviour, and fault response. It was observed that Mesh topology offers the highest fault tolerance while Bus topology is most vulnerable to failures. Star topology provides the best balance of cost and manageability for typical LANs, while Hybrid topology is preferred in real-world enterprise deployments due to its flexibility and resilience.

About This Lab

Overview

This virtual lab was developed as part of the Computer Networks Laboratory curriculum for undergraduate engineering students. It provides an interactive, browser-based simulation of the six fundamental network topologies — no software installation required.

The simulation visualises node and link structure, animates packet transmission paths, and demonstrates fault behaviour — helping students build intuition about real-world network design trade-offs.

Technologies Used

  • HTML5 Canvas — for drawing topology diagrams and packet animations
  • Vanilla JavaScript — full topology engine, packet routing, and fault simulation with no dependencies
  • CSS3 — responsive dark-theme layout matching the DVR Lab design language
  • Google Fonts — Syne, JetBrains Mono, Inter
