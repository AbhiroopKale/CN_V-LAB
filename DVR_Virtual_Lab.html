Theory

1. What is Routing?

Routing is the process of selecting a path for traffic in a network from source to destination. A router is a network device that forwards data packets between networks based on its routing table — a data structure stored in memory that maps destination networks to outgoing links and associated costs.

Routing algorithms are classified into two types:

  • Static Routing — routes are manually configured by an administrator. Simple, but does not adapt to network changes.
  • Dynamic Routing — routers automatically discover paths and update tables using algorithms. Adapts to topology changes in real time.

2. Distance Vector Routing

Distance Vector Routing (DVR) is a dynamic routing algorithm based on the principle that each router maintains a table (the distance vector) listing the best-known distance to each destination and the interface to use. This table is initialised with directly connected networks and updated iteratively.

Key characteristics:

  • Each router only knows about its direct neighbours — not the full network topology.
  • Routers periodically share their entire routing table with all neighbours ("routing by rumour").
  • On receiving a neighbour's table, a router computes new distances and updates its own table if a shorter path is found.
  • The algorithm converges when no router's table changes in a round.

3. The Bellman-Ford Equation

The mathematical backbone of DVR is the Bellman-Ford equation. For a router X trying to reach destination D via neighbour Y:

dX(D)  =  minY ∈ neighbours(X) { cost(X, Y)  +  dY(D) }

Where:

  • dX(D) = best-known distance from router X to destination D
  • cost(X, Y) = cost of the direct link between X and its neighbour Y (usually 1 hop)
  • dY(D) = distance as reported by neighbour Y to reach D

Example: Router A wants to reach Net-55. Neighbour B says it can reach Net-55 in 1 hop. The link A→B costs 1 hop. So A computes: dA(55) = 1 + 1 = 2 hops via B.

4. How the Algorithm Works — Step by Step

  • Initialisation: Each router fills its table with directly connected networks (cost = 1 hop each). All other destinations are unknown (∞).
  • Exchange: Every router sends a copy of its routing table to all directly connected neighbours.
  • Update: Each router receives neighbour tables, adds 1 hop to each entry (for the link cost), and updates its own table where a shorter path is found.
  • Repeat: Exchange and update continue in rounds until no changes occur.
  • Convergence: The network is converged when all routers have stable, consistent routing tables with optimal paths to all destinations.

5. Problems with DVR

  • Slow Convergence: In large networks, it takes many rounds for information to propagate — routers far apart are slow to learn new routes.
  • Count-to-Infinity: When a link fails, routers may loop indefinitely, incrementing the distance to an unreachable destination. This is typically bounded by setting a maximum hop count (e.g., 15 in RIP).
  • Routing Loops: Two routers may route packets to each other, forming a loop, until the hop count reaches infinity.

6. Real-World Protocol: RIP

The Routing Information Protocol (RIP) is the most well-known implementation of Distance Vector Routing. Originally used in ARPANET, it remains in use today in small networks.

  • Uses hop count as the metric (max 15 hops; 16 = infinity/unreachable).
  • Sends routing updates every 30 seconds to all neighbours.
  • Uses split horizon and poison reverse to mitigate routing loops.
  • Available as RIP v1 (classful) and RIP v2 (classless, supports VLSM and CIDR).

Open the Simulation Tab

Navigate to the Simulation tab in this virtual lab. You will see the network topology with 6 routers (A–F) displayed as nodes connected by labelled edges representing networks.

Inspect Initial Routing Tables (Round 0)

Click on each router individually to view its routing table in the right panel. At Round 0, each router only knows its directly connected networks with a cost of 1 hop.

Record: Write down Router A's initial table. Which networks does it know? What is the next hop for each?
3

Run Round 1

Click "▶ Run Round" button. Each router exchanges its table with all direct neighbours. Watch routers flash amber — this indicates their tables were updated with new information.

Observe: How many new routes did Router A learn? What is the cost and next hop for each new entry? Notice that entries highlighted in green are newly learned routes.
4

Continue Running Rounds

Keep clicking "Run Round" and click on Router A (or any router) after each round to inspect its updated table. Use the Auto-Run button to animate the process automatically.

Record: After each round, note how many total routes each router knows. Fill in your observation table.
5

Observe Convergence

Continue running rounds until the log panel shows "Tables converged — no changes this round". The convergence progress bar will also reach 100%.

Record: How many rounds did it take to fully converge? What is the final routing table of each router?
6

Verify Using Bellman-Ford

Manually compute the path from Router A to Network 66 using the Bellman-Ford equation. Trace: A → ? → ? → Net-66. Verify your manual calculation matches the simulation result.

Expected: A reaches Net-66 in 3 hops via B (A→B→C→Net-66) or via E→D (A→E→D→Net-66). Both paths cost 3 hops.
7

Reset and Repeat

Click Reset and repeat the experiment. This time, focus on a different router (e.g., Router F which is centrally placed) and trace how it learns routes.

About This Lab

Overview

This virtual lab was designed as part of the Computer Networks Laboratory curriculum for undergraduate engineering students. It provides an interactive, browser-based simulation of the Distance Vector Routing algorithm without requiring any software installation.

The simulation faithfully models the exchange of routing tables between routers in a 6-node network, demonstrating iterative convergence through the Bellman-Ford algorithm — the same mechanism used in the real-world Routing Information Protocol (RIP).

Technologies Used

  • HTML5 Canvas — for drawing the network topology and animations
  • Vanilla JavaScript — full DVR algorithm implementation with no dependencies
  • CSS3 — responsive dark-theme layout with transitions
  • Google Fonts — Syne, JetBrains Mono, Inter
