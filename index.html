What is Selective Repeat ARQ?

Selective Repeat ARQ is a sliding window protocol used in data link and transport layers. Unlike Go-Back-N, it retransmits only the specific frames that are lost or corrupted — allowing the receiver to buffer out-of-order frames and deliver them in sequence.

Selective Repeat ARQ operates as follows:

  • The sender maintains a window of size Ws and can transmit up to Ws unacknowledged frames.
  • The receiver also maintains a window of size Wr and can accept out-of-order frames, buffering them.
  • Each frame is individually acknowledged. A NACK or timeout causes only that specific frame to be retransmitted.
  • Buffered frames are delivered to the upper layer in order once all gaps are filled.
💡 Key insight: The receiver does not discard out-of-order frames. It stores them in a buffer until the missing frame arrives, then delivers a contiguous block.

The sliding window mechanism:

  • Sender Window: Frames from base to base + Ws − 1 can be in-flight simultaneously.
  • The window "slides" forward only when the frame at the base (lowest unacknowledged) receives its ACK.
  • Receiver Window: Accepts frames within [rcv_base, rcv_base + Wr − 1].
  • Out-of-window frames are discarded.
  • Sequence numbers range from 0 to 2n − 1, where n is the number of bits used.
  • For Selective Repeat: Ws = Wr ≤ 2n−1
  • If the window were larger, the receiver could confuse new frames with retransmitted ones.
  • Example: 3-bit sequence → sequence numbers 0–7 → max window = 4
⚠️ Rule: Ws + Wr ≤ 2n (for Selective Repeat, both windows equal, so W ≤ 2n/2 = 2n−1)
